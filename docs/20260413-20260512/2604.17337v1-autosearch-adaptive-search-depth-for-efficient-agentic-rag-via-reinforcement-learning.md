## 摘要
智能体检索增强生成（Agentic RAG）系统使大语言模型（LLM）能够通过与外部检索工具的多步交互来解决复杂任务。然而，这种多步交互通常涉及冗余的搜索步骤，从而产生巨大的计算成本和延迟。先前的工作通过限制搜索深度（即搜索步数）来降低成本，但这往往导致对复杂问题的探索不足。为了解决这一问题，我们首先研究了搜索深度如何影响准确性，并发现了一个定义准确率与效率权衡的最小充分搜索深度，该深度由问题复杂度和智能体能力共同决定。此外，我们提出了 AutoSearch，这是一个强化学习（RL）框架，通过自生成的中间答案来评估每个搜索步骤。通过自回答机制，AutoSearch 识别出最小充分搜索深度，并通过奖励达到该深度同时惩罚过度搜索来促进高效搜索。此外，还引入了奖励机制以稳定搜索行为并提高复杂问题的回答质量。在多个基准测试上的广泛实验表明，AutoSearch 实现了卓越的准确率与效率权衡，在保持搜索质量的同时缓解了过度搜索。

## Abstract
Agentic retrieval-augmented generation (RAG) systems enable large language models (LLMs) to solve complex tasks through multi-step interaction with external retrieval tools. However, such multi-step interaction often involves redundant search steps, incurring substantial computational cost and latency. Prior work limits search depth (i.e., the number of search steps) to reduce cost, but this often leads to underexploration of complex questions. To address this, we first investigate how search depth affects accuracy and find a minimal sufficient search depth that defines an accuracy-efficiency trade-off, jointly determined by question complexity and the agent's capability. Furthermore, we propose AutoSearch, a reinforcement learning (RL) framework that evaluates each search step via self-generated intermediate answers. By a self-answering mechanism, AutoSearch identifies the minimal sufficient search depth and promotes efficient search by rewarding its attainment while penalizing over-searching. In addition, reward mechanisms are introduced to stabilize search behavior and improve answer quality on complex questions. Extensive experiments on multiple benchmarks show that AutoSearch achieves a superior accuracy-efficiency trade-off, alleviating over-searching while preserving search quality.

*   **核心问题**：如何让智能体根据“问题的复杂度”和“自身的理解能力”，自适应地找到**最小充分搜索深度**，从而在保证准确率的前提下实现效率最大化。

### 2. 方法论
*   **核心思想**：利用强化学习（RL）训练模型，使其在每一步搜索后尝试生成“中间答案”。通过观察中间答案何时达到正确，来定义该问题的最优搜索深度，并以此作为奖励信号。
*   **关键技术细节**：
    *   **自回答机制**：在每一个检索步骤 $t$，模型不仅决定下一步搜什么，还会被提示生成一个中间答案 $a_{inter}$。
    *   **能力感知最优深度 ($t_c$)**：定义为模型第一次生成正确中间答案的最小步数。
    *   **奖励函数设计**：
        *   **基础奖励 (Base Reward)**：确保输出格式正确，并对最终答案的正确性（EM）给予奖励。
        *   **搜索效率奖励 (Efficiency Reward)**：这是核心。如果模型在 $t_c$ 步之前停止，给予正向引导；如果超过 $t_c$ 步仍在搜索，则给予负向惩罚。
        *   **搜索质量奖励 (Quality Reward)**：计算当前步中间答案相对于前一步在 F1 分数上的增益，奖励那些能提供实质性新信息的搜索行为。
*   **算法流程**：采用 PPO（近端策略优化）作为主要的 RL 算法，同时也验证了 GRPO（群体相对策略优化）的适用性。

### 6. 主要结论与发现
*   **自适应性**：AutoSearch 能够根据问题难度自动调整深度。在简单问题（单跳）上保持极低深度，在复杂问题（多跳）上则自动增加步数。

*   **能力关联**：实验发现，模型越强（如 7B vs 3B），其所需的最小充分搜索深度往往越浅，因为强模型对信息的整合能力更强。

### 7. 优点
*   **闭环自评估**：通过中间答案让模型“自省”搜索是否足够，摆脱了对固定步数或外部昂贵模型（如 GPT-4）评判的依赖。
*   **多维奖励平衡**：巧妙地结合了格式、结果、效率和质量四个维度的奖励，解决了 RL 训练中奖励稀疏的问题。
*   **实用性强**：在小参数模型（3B/7B）上取得了显著效果，非常适合需要低延迟、低成本的工业级 RAG 应用。

### 8. 不足与局限
*   **搜索深度上限**：实验中设定的最大搜索步数相对较小（通常为 4 步），对于极度复杂的长程推理任务（如需要 10 步以上搜索的任务）尚未验证。
*   **依赖 Ground Truth**：训练过程仍需要标准答案来计算 $t_c$，在完全无监督或开放域实时任务中的在线学习能力有待探索。
*   **检索器固定**：实验主要关注 LLM 的决策能力，未探讨检索器（Retriever）本身的性能波动对自适应深度的影响。
