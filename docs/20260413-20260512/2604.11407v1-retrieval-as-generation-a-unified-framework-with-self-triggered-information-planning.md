## 摘要
我们通过将检索控制直接嵌入到生成过程中，重新审视了检索增强生成（RAG）。我们不再将检索视为外部干预，而是在词元级解码中表达检索决策，从而在无需额外控制器或分类器的情况下实现端到端协作。在“检索即生成”的范式下，我们提出了 GRIP（生成引导的信息规划检索），这是一个统一的框架，模型通过发射控制词元来调节检索行为。GRIP 的核心是“自触发信息规划”，它允许模型在单个自回归轨迹中决定何时检索、如何重构查询以及何时终止。这种设计将检索与推理紧密耦合，并支持带有即时证据集成的动态多步推理。为了监督这些行为，我们构建了一个结构化训练集，涵盖了可回答、部分可回答和多跳查询，每种查询都与特定的词元模式对齐。在五个问答基准测试上的实验表明，GRIP 超越了强大的 RAG 基准模型，并且在参数量大幅减少的情况下，其性能可与 GPT-4o 媲美。

## Abstract
We revisit retrieval-augmented generation (RAG) by embedding retrieval control directly into generation. Instead of treating retrieval as an external intervention, we express retrieval decisions within token-level decoding, enabling end-to-end coordination without additional controllers or classifiers. Under the paradigm of Retrieval as Generation, we propose \textbf{GRIP} (\textbf{G}eneration-guided \textbf{R}etrieval with \textbf{I}nformation \textbf{P}lanning), a unified framework in which the model regulates retrieval behavior through control-token emission. Central to GRIP is \textit{Self-Triggered Information Planning}, which allows the model to decide when to retrieve, how to reformulate queries, and when to terminate, all within a single autoregressive trajectory. This design tightly couples retrieval and reasoning and supports dynamic multi-step inference with on-the-fly evidence integration. To supervise these behaviors, we construct a structured training set covering answerable, partially answerable, and multi-hop queries, each aligned with specific token patterns. Experiments on five QA benchmarks show that GRIP surpasses strong RAG baselines and is competitive with GPT-4o while using substantially fewer parameters.

*   **整体含义**：论文提出了 **GRIP** 框架，倡导“检索即生成”（Retrieval as Generation）的范式，将检索控制直接嵌入到词元（Token）级的解码过程中，使模型能够自主、端到端地规划检索行为。

### 2. 方法论
*   **核心思想**：通过在模型的输出空间中引入特定的**控制词元（Control Tokens）**，将检索触发、查询重构和终止决策转化为自回归生成任务的一部分。
*   **关键技术细节**：
    *   **控制词元集**：引入了 `[RETRIEVE]`（触发检索）、`[INTERMEDIARY]`（输出中间状态）、`[ANSWER]`（开始最终回答）和 `[SOLVED]`（终止生成）。
    *   **自触发信息规划**：模型在生成过程中自主判断内部知识是否充足。若不足，则发射 `[INTERMEDIARY]` 和 `[RETRIEVE]`，并随后生成针对性的新查询语句。
    *   **两阶段训练方案**：
        1.  **结构化监督微调 (SFT)**：构建了四种类型的训练数据（Type-α 直接回答、Type-β 需检索、Type-γ 多跳规划、Type-θ 信息整合），涵盖 40,000 条样本。
        2.  **规则引导的强化学习 (RL)**：使用 DAPO 算法（GRPO 的变体），通过“答案忠实度奖励”（基于 BLEU）和“控制准确性奖励”来优化模型，减少过度检索并提升回答质量。

*   **数据集**：涵盖了 5 个主流问答基准：**HotpotQA**（多跳推理）、**PopQA**、**Natural Questions (NQ)**、**WebQuestions (WebQ)** 和 **TriviaQA**。此外还在生物医学领域数据集 **BioASQ** 上进行了领域迁移测试。

*   **Benchmark 设置**：统一使用 LLaMA3-8B 作为骨干模型，Wikipedia 作为检索库，BM25 作为默认检索器。

    *   **能力保持测试**：在 MMLU 和 MBPP 上测试了模型在引入检索规划能力后，是否保持了通用的知识和编程能力。

### 6. 主要结论与发现
*   **性能卓越**：GRIP 在所有基准上均显著优于现有的开源 RAG 框架，且在参数量远小于 GPT-4o 的情况下，表现出极强的竞争力。
*   **检索深度自适应**：GRIP 能根据任务难度自动调整检索次数。例如在多跳任务（HotpotQA）中检索较多，而在简单事实任务（NQ）中检索较少。
*   **查询质量提升**：模型生成的重构查询比原始查询能更有效地覆盖正确答案（Top-1/3 覆盖率显著提升）。
*   **外推能力**：虽然训练时最大检索步数为 3，但在推理时增加预算（如 B=10），模型仍能通过更多步检索持续提升性能。

### 7. 优点
*   **高度统一性**：无需外部分类器或复杂的 Agent 框架，检索逻辑完全内化在词元生成中。
*   **可解释性强**：通过控制词元的发射，可以清晰地观察到模型的推理路径和检索动机。
*   **效率与效果平衡**：相比于 R1-Searcher 等深度搜索方法，GRIP 以更少的检索次数达到了更高的准确率。

### 8. 不足与局限
*   **接口敏感性**：模型表现仍受限于检索器的性能（如 BM25 的局限）和文档分块策略。
*   **训练成本**：虽然推理高效，但需要构建高质量的结构化轨迹数据进行 SFT 和 RL 训练，对数据工程依赖较高。
*   **冗余风险**：在某些情况下，模型仍可能出现不必要的重复检索，尽管 RL 阶段已有所缓解。
