## 摘要
检索增强生成（RAG）通常将检索和生成视为独立的系统。我们探讨基于注意力的编码器-解码器是否可以直接从其自身的内部表示中进行检索。我们引入了 INTRA（通过注意力的内在检索），这是一个解码器注意力查询对预编码的证据块进行评分，并将其直接重用为生成上下文的框架。通过这种构建方式，INTRA 统一了检索和生成，消除了 RAG 流水中典型的检索器-生成器不匹配问题。该设计还通过在不同查询之间重用预计算的编码器状态，分摊了上下文编码的开销。在问答基准测试中，INTRA 在证据召回率和端到端回答质量方面均优于强大的工程化检索流水线。我们的结果表明，基于注意力的模型已经具备了一种可以被激发的检索机制，而不是作为外部模块添加。

## Abstract
Retrieval-augmented generation (RAG) typically treats retrieval and generation as separate systems. We ask whether an attention-based encoder-decoder can instead retrieve directly from its own internal representations. We introduce INTRA (INTrinsic Retrieval via Attention), a framework where decoder attention queries score pre-encoded evidence chunks that are then directly reused as context for generation. By construction, INTRA unifies retrieval and generation, eliminating the retriever-generator mismatch typical of RAG pipelines. This design also amortizes context encoding by reusing precomputed encoder states across queries. On question-answering benchmarks, INTRA outperforms strong engineered retrieval pipelines on both evidence recall and end-to-end answer quality. Our results demonstrate that attention-based models already possess a retrieval mechanism that can be elicited, rather than added as an external module.

### 1. 核心问题与研究动机
*   **核心问题**：传统的检索增强生成（RAG）将“检索”和“生成”视为两个独立的模块，这导致了两个主要问题：
    1.  **表示空间不匹配**：检索器（如双编码器）和生成器（LLM）在不同的向量空间运行，检索到的内容不一定最符合生成器的需求。
    2.  **计算冗余**：检索到的文本在生成阶段需要被生成器重新编码（Re-encoding），造成了巨大的计算开销。
*   **研究动机**：作者认为“注意力机制”本质上就是一种基于查询的信息选择机制。因此，他们探讨是否可以利用模型内部的交叉注意力（Cross-Attention）直接从语料库的内部表示中进行检索，从而实现检索与生成的深度统一。

### 2. 方法论：INTRA 框架
*   **核心思想**：将检索过程转化为解码器对预编码（Pre-encoded）证据块的注意力评分过程。
*   **关键技术细节**：
    *   **内在检索 (Intrinsic Retrieval)**：利用解码器的交叉注意力查询向量（Query）与编码器生成的块表示（Key）进行匹配。
    *   **Reverse-QWK (逆向查询-键投影)**：为了让所有解码器层能共享同一个编码器表示池，作者将原本位于 Key 侧的层特定投影矩阵和缩放因子移到了 Query 侧。这样，语料库只需编码一次并存储为归一化状态，即可供所有层检索使用。
    *   **可学习检索 Token**：在解码器输入中加入专门的检索 Token（Retrieval Tokens），通过训练这些 Token 的嵌入来优化检索查询。
    *   **MaxSim 评分**：借鉴 ColBERT，通过计算查询 Token 与块 Token 之间的最大相似度之和来对语料块进行排名。
    *   **池化表示**：为了降低检索计算量，将每个块的 Token 序列平均池化为较短的序列（如长度为 7），在保留模型内部特征的同时提高匹配效率。

*   **客观性**：通过固定生成器来对比不同检索方法，并引入了“Gap Closure”（间隙闭合率）指标来衡量检索结果对最终答案的贡献，实验设计严谨、公平。

### 6. 主要结论与发现
*   **多跳任务表现卓越**：INTRA 在多跳问答上的表现显著优于专门训练的外部检索器（如 Qwen3-Emb），证明了注意力机制在处理复杂证据链方面的内在优势。
*   **统一表示的红利**：当检索器和生成器共享同一个解码器逻辑时，证据的筛选与生成需求高度对齐，闭合了性能间隙。
*   **显著的效率提升**：由于无需在生成阶段重新编码检索到的文本，INTRA 的首字延迟（TTFT）比标准 RAG 快数倍甚至数十倍（随检索块数量增加而优势扩大）。

### 7. 优点
*   **架构创新**：打破了 RAG 模块化的思维定式，证明了模型“内部”就蕴含检索能力。
*   **计算高效**：通过 Reverse-QWK 实现了编码器状态的完全复用，极大地降低了推理成本。
*   **端到端一致性**：解决了检索与生成之间“所选非所需”的表示失配问题。

### 8. 不足与局限
*   **架构受限**：该方法高度依赖 Encoder-Decoder 架构的交叉注意力，目前无法直接应用于纯解码器（Decoder-only）模型（如 GPT-4, Llama）。
*   **存储压力**：虽然减少了计算，但存储 Token 级别的向量表示（即使经过池化）比存储原始文本或单向量索引需要更多的磁盘/内存空间。
*   **检索规模**：实验语料库规模为亿级 Token，在处理万亿级超大规模网页检索时的索引构建和检索延迟仍需进一步验证。
