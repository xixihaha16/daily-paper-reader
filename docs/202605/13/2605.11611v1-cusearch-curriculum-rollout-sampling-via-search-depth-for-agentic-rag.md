## 摘要
具有可验证奖励的强化学习（RLVR）已成为一种仅通过结果监督来训练智能体检索增强生成（RAG）系统的极具前景的范式。大多数现有方法通过均匀采样的展开（rollouts）来优化策略，隐含地将所有轨迹视为具有同等的信息量。然而，轨迹在搜索深度上存在显著差异，因此其信息量并不相等：搜索深度较深的轨迹包含更多的检索决策点，并为检索子策略提供更密集的直接监督。此外，随着训练的进行，批次内的深度分布向更高值偏移，这种异质性也随之增加，但均匀展开采样对这种偏移视而不见。为解决这一问题，我们提出了 CuSearch，这是一个基于搜索深度贪婪分配（SDGA）的课程展开采样框架。SDGA 是一种批次级算子，可将固定的更新预算重新分配给搜索深度较深的轨迹。SDGA-Auto 始终针对当前批次中最深的可用轨迹，随着深度分布的上移，产生一种隐式的、与训练对齐的课程。SDGA-Phase 则在较深轨迹变得足够丰富时显式地推进课程阈值。跨模型类型和检索框架的实验表明，CuSearch 持续提升了性能，在 ZeroSearch 上比标准 GRPO 提高了多达 11.8 个精确匹配点。这些结果表明，在基于 RLVR 的智能体 RAG 训练中，单条轨迹的搜索深度是检索监督密度的一种可靠且无需标注的代理指标。代码可在 https://github.com/MrToser/CuSearch 获取。

## Abstract
Reinforcement Learning with Verifiable Rewards (RLVR) has emerged as a promising paradigm for training agentic retrieval-augmented generation (RAG) systems from outcome-only supervision. Most existing methods optimize policies from uniformly sampled rollouts, implicitly treating all trajectories as equally informative. However, trajectories differ substantially in search depth and are therefore not equally informative: deeper-search trajectories contain more retrieval decision points and provide denser direct supervision for the retrieval sub-policy. Moreover, this heterogeneity grows over training as the within-batch depth distribution shifts toward higher values, yet uniform rollout sampling remains blind to this shift. To address this, we propose CuSearch, a curriculum rollout sampling framework built on Search-Depth Greedy Allocation (SDGA), a batch-level operator that reallocates a fixed update budget toward deeper-search trajectories. SDGA-Auto always targets the deepest available trajectories in the current batch, yielding an implicit training-aligned curriculum as the depth distribution shifts upward. SDGA-Phase explicitly advances the curriculum threshold as deeper trajectories become sufficiently abundant. Experiments across model types and retrieval frameworks show that CuSearch consistently improves performance, achieving up to 11.8 exact-match points over standard GRPO on ZeroSearch. These results establish per-trajectory search depth as a reliable, annotation-free proxy for retrieval supervision density in RLVR-based agentic RAG training. The code is available at https://github.com/MrToser/CuSearch.

*   **核心问题**：现有的方法（如 GRPO）通常对所有生成的轨迹（Rollouts）进行**均匀采样**。然而，不同的轨迹包含的检索决策点数量不同。
*   **研究动机**：作者发现“搜索深度”（即一条轨迹中有效搜索的次数）是衡量检索策略监督信号密度的关键指标。随着训练进行，模型生成的搜索深度会自然增加，但均匀采样忽略了这种分布偏移，导致计算预算被浪费在缺乏检索信息的浅层轨迹上。

### 2. 方法论
CuSearch 的核心是 **搜索深度贪婪分配（SDGA）** 算子，它在批次级别将固定的更新预算 $K$ 重新分配给搜索深度更深的轨迹。
*   **核心思想**：将生成的轨迹按搜索次数分桶（Buckets），优先选择深层桶中的轨迹进行梯度更新。
*   **关键技术细节**：
    *   **SDGA-Auto（隐式课程）**：无状态的贪婪策略，始终优先选择当前批次中最深的轨迹。随着模型能力的提升，选中的轨迹深度会自然上移。
    *   **SDGA-Phase（显式课程）**：维护一个显式的阶段阈值，只有当更深的轨迹在采样池中足够丰富时，才不可逆地推进到下一阶段。
*   **算法流程**：
    1.  采样生成 $N \times G$ 条轨迹。
    2.  解析每条轨迹的搜索次数并分桶。
    3.  根据优先级（深度优先）和桶容量，贪婪地分配 $K$ 个更新名额。
    4.  仅对选中的 $K$ 条轨迹计算优势（Advantage）并更新策略。

### 6. 主要结论与发现
*   **性能提升显著**：在 ZeroSearch 框架下，CuSearch 相比标准 GRPO 在 Qwen2.5-3B 上提升了 **11.8** 个 EM 点。
*   **搜索深度是优质代理指标**：搜索深度是一个无需标注、轨迹固有的结构化信号，能有效反映检索监督密度。
*   **环境随机性的影响**：检索环境的随机性（Stochasticity）会放大 CuSearch 的收益，因为高随机性环境下每一步搜索带来的新信息更多。
*   **奖励并非最佳采样依据**：实验发现，仅按奖励高低筛选轨迹（Top-K Reward）效果不如按搜索深度筛选，因为高奖励轨迹可能完全没有搜索行为，无法提供检索子策略的训练信号。

### 7. 优点
*   **无需额外标注**：不需要过程监督（Process Supervision）或昂贵的专家轨迹，完全依赖轨迹自身的结构特征。
*   **即插即用**：作为轻量级的轨迹选择层，无需修改奖励函数、模型架构或环境。
*   **理论支撑**：附录中通过策略梯度分解，证明了最大化搜索深度等同于最大化检索梯度的覆盖范围。
*   **计算效率高**：在相同的 Rollout 预算下，通过更精细的样本选择实现了更高的性能。

### 8. 不足与局限
*   **环境依赖性**：在搜索深度增长缓慢或检索环境高度确定（如重复查询返回完全相同结果）的情况下，增益会收缩。

*   **任务覆盖**：目前主要集中在开放域问答（QA），尚未验证在工具调用（Tool Use）或代码执行等其他智能体行为中的泛化能力。
