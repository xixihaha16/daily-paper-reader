## 摘要
单步检索增强生成（RAG）为简单问答任务提供了一种整合外部信息的有效方式，但在处理复杂问题时表现不佳。智能体 RAG（Agentic RAG）扩展了这一范式，将单步检索替换为多步过程，其中大语言模型（LLM）充当搜索智能体，生成中间思维和子查询，以与检索系统进行迭代交互。由于冗长思维和子查询的自回归生成，这种迭代过程会产生显著的延迟。为了解决这一局限性，我们提出了 LatentRAG，这是一个将推理和检索从离散语言空间转移到连续潜空间的新型框架。与现有的逐标记生成自然语言思维或子查询的显式方法不同，LatentRAG 在单次前向传播中直接从隐藏状态生成思维和子查询的潜标记（latent tokens）。我们在潜空间中将 LLM 与稠密检索模型对齐，实现了基于潜子查询标记的检索，并支持端到端的联合优化。为了提高透明度并促进具有语义意义的潜表示，我们引入了一种并行潜解码机制，将潜标记翻译回自然语言。在七个基准数据集上的广泛实验表明，LatentRAG 达到了与显式智能体 RAG 方法相当的性能，同时将推理延迟降低了约 90%，大幅缩小了与传统单步 RAG 的延迟差距。

## Abstract
Single-step retrieval-augmented generation (RAG) provides an efficient way to incorporate external information for simple question answering tasks but struggles with complex questions. Agentic RAG extends this paradigm by replacing single-step retrieval with a multi-step process, in which the large language model (LLM) acts as a search agent that generates intermediate thoughts and subqueries to iteratively interact with the retrieval system. This iterative process incurs substantial latency due to the autoregressive generation of lengthy thoughts and subqueries. To address this limitation, we propose LatentRAG, a novel framework that shifts both reasoning and retrieval from discrete language space to continuous latent space. Unlike existing explicit methods that generate natural language thoughts or subqueries token-by-token, LatentRAG produces latent tokens for thoughts and subqueries directly from the hidden states in a single forward pass. We align LLMs with dense retrieval models in the latent space, enabling retrieval over latent subquery tokens and supporting end-to-end joint optimization. To improve transparency and encourage semantically meaningful latent representations, we incorporate a parallel latent decoding mechanism that translates latent tokens back into natural language. Extensive experiments on seven benchmark datasets show that LatentRAG achieves performance comparable to explicit agentic RAG methods while reducing inference latency by approximately 90%, substantially narrowing the latency gap with traditional single-step RAG.

#### 1. 核心问题与整体含义（研究动机和背景）
传统的**智能体 RAG（Agentic RAG）**通过让大语言模型（LLM）生成中间思维（Thoughts）和子查询（Subqueries）来处理复杂的多步问答任务。然而，这种方法存在严重的**延迟瓶颈**：由于思维和子查询是逐标记（token-by-token）自回归生成的，其推理时间通常是传统单步 RAG 的 16-22 倍，其中 90% 的时间消耗在文本生成阶段。

本文提出了 **LatentRAG**，旨在将推理和检索过程从离散的“语言空间”转移到连续的“潜空间（Latent Space）”。其核心动机是在不显著损失性能的前提下，通过单次前向传播生成潜表示，大幅提升 Agentic RAG 的运行效率。

#### 2. 方法论：核心思想与关键技术
LatentRAG 的核心思想是利用 LLM 的隐藏状态（Hidden States）直接进行推理和检索，避免冗长的文本解码。

*   **潜标记生成（Generation with Latent Tokens）**：
    在输入中插入特殊的思维标记（`<think>`）和查询标记（`<query>`）。LLM 不再逐个生成单词，而是通过单次前向传播，利用这些特殊标记对应的最后一层隐藏状态作为“潜思维”和“潜子查询”。
*   **潜空间检索（Latent Retrieval）**：
    引入一个轻量级的投影层（Projector），将 LLM 的潜子查询标记映射到检索模型的嵌入空间。为了解决数据稀缺问题，模型通过 **KL 散度损失** 进行对齐训练，使潜子查询产生的文档分布逼近自然语言查询产生的分布。
*   **并行潜解码（Parallel Latent Decoding）**：
    为了保证透明度，LatentRAG 包含一个可选的解码分支，将潜标记翻译回自然语言。由于各步骤的潜标记已生成，所有思维和子查询可以**并行解码**，而非串行生成。
*   **联合训练目标**：
    训练目标结合了生成损失（$L_{gen}$）、检索对齐损失（$L_{ret}$）和潜解码损失（$L_{dec}$），支持端到端的优化。

#### 5. 实验数量与充分性
实验设计非常充分且具有说服力：
*   **多维度评估**：在 7 个数据集上进行了全面测试，并针对 5 种不同的检索模型验证了通用性。
*   **消融实验**：验证了 KL 损失、预训练检索模型、潜解码损失等组件的必要性。
*   **扩展性分析**：研究了 LLM 规模（3B 到 14B）和检索模型规模（0.6B 到 8B）对性能和延迟的影响。
*   **客观性**：使用了基准方法的公开权重进行对比，并详细拆解了每一阶段（预填充、生成、检索、回答）的延迟。

#### 6. 主要结论与发现
*   **极高的效率提升**：LatentRAG 将 Agentic RAG 的推理延迟降低了约 **90%**，使其响应速度接近传统的单步 Naive RAG。
*   **性能保持**：在大幅提速的同时，其准确率（EM 分数）与最强的显式方法（如 Search-R1）相比，差距通常在 **5% 以内**，甚至在某些设置下有所超越。
*   **并行优势**：即使开启潜解码以保证透明度，由于支持跨步骤并行处理，仍能比显式方法快 40%-60%。
*   **对齐挑战**：发现检索模型的各向异性（Anisotropy）会影响对齐效果，例如 e5 模型由于嵌入分布过于集中，导致性能下降较明显。

#### 7. 优点
*   **创新性强**：首次将潜推理（Latent Reasoning）与智能体检索（Agentic Retrieval）结合，解决了 RAG 智能体落地的核心痛点——慢。
*   **端到端优化**：通过潜空间对齐，使 LLM 和检索器能够通过梯度进行联合优化，而非仅仅依赖提示词。
*   **灵活性**：提供了效率（纯潜空间运行）与透明度（并行解码）之间的权衡选项。

#### 8. 不足与局限
*   **依赖教师模型**：目前的训练依赖于显式 Agentic RAG 生成的轨迹（SFT），性能上限受限于教师模型的质量。
*   **精确度风险**：案例分析显示，潜表示在处理极其精确的词汇（如人名拼写）时，偶尔会出现微小偏差（如将 Chilcot 拼成 Chilcott）。
*   **检索模型敏感性**：对检索模型的嵌入空间几何特性有一定要求，某些高度压缩或分布不均的检索模型可能难以对齐。
