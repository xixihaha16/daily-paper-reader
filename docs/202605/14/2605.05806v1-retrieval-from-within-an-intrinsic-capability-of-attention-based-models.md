## 摘要
检索增强生成（RAG）通常将检索和生成视为独立的系统。我们探讨了基于注意力机制的编码器-解码器是否可以直接从其自身的内部表示中进行检索。我们引入了 INTRA（通过注意力的内在检索），这是一个解码器注意力查询对预编码的证据块进行评分，并将其直接重用为生成上下文的框架。通过这种构建方式，INTRA 统一了检索和生成，消除了 RAG 流水线中典型的检索器-生成器不匹配问题。该设计还通过在不同查询中重用预计算的编码器状态，分摊了上下文编码的开销。在问答基准测试中，INTRA 在证据召回率和端到端回答质量方面均优于强大的工程化检索流水线。我们的结果表明，基于注意力机制的模型已经具备了一种可以被激发出来的检索机制，而不是必须作为一个外部模块添加。

## Abstract
Retrieval-augmented generation (RAG) typically treats retrieval and generation as separate systems. We ask whether an attention-based encoder-decoder can instead retrieve directly from its own internal representations. We introduce INTRA (INTrinsic Retrieval via Attention), a framework where decoder attention queries score pre-encoded evidence chunks that are then directly reused as context for generation. By construction, INTRA unifies retrieval and generation, eliminating the retriever-generator mismatch typical of RAG pipelines. This design also amortizes context encoding by reusing precomputed encoder states across queries. On question-answering benchmarks, INTRA outperforms strong engineered retrieval pipelines on both evidence recall and end-to-end answer quality. Our results demonstrate that attention-based models already possess a retrieval mechanism that can be elicited, rather than added as an external module.

*   **研究动机**：作者认为“注意力机制”本质上就是一种查询驱动的选择机制。因此，他们探讨是否可以利用预训练编码器-解码器模型（Encoder-Decoder）**内在的注意力机制**直接从其内部表示中检索信息，从而实现检索与生成的深度统一。

### 2. 方法论：INTRA 框架
*   **核心思想**：将检索过程转化为解码器对预编码证据块（Evidence Chunks）的交叉注意力评分过程。
*   **关键技术细节**：
    *   **统一表示空间**：使用同一个模型的编码器对语料库进行预编码，存储这些隐藏状态（Latent Representations）而非原始文本。
    *   **检索 Token（Retrieval Tokens）**：在解码器输入中加入少量可学习的特殊 Token，专门用于生成检索查询。
    *   **MaxSim 评分**：借鉴 ColBERT 的后期交互机制，利用解码器查询向量与编码器块向量的相似度进行评分，选出 Top-k 个块。
    *   **Reverse-QWK（反向查询-键投影）**：这是本文的技术亮点。为了让所有解码器层能共享同一个编码器表示索引，作者将原本属于 Key 侧的线性投影和缩放参数移到了 Query 侧，从而实现跨层的高效检索。
    *   **状态复用**：检索到的块表示直接作为解码器的交叉注意力上下文，无需重新编码文本，显著降低了推理延迟。

### 6. 主要结论与发现
*   **检索性能**：INTRA 在多跳问答任务上表现极佳，其证据召回率显著超过了专门训练的外部检索器。
*   **生成质量**：由于检索与生成在同一表示空间内对齐，INTRA 在端到端回答准确度上优于所有基准。
*   **效率优势**：通过复用预编码状态，INTRA 避免了 RAG 中昂贵的文本重编码过程，大幅降低了首字延迟（TTFT）。

### 7. 优点
*   **架构优雅**：消除了检索器和生成器之间的“翻译”代价，实现了真正的端到端对齐。
*   **推理高效**：Reverse-QWK 技术解决了多层 Transformer 检索的工程难题，支持预计算和状态复用。
*   **多跳能力强**：相比单向量检索，基于 Token 级注意力的 INTRA 更擅长处理复杂的证据组装任务。

### 8. 不足与局限
*   **架构限制**：该方法依赖于编码器-解码器（Encoder-Decoder）架构，无法直接应用于目前主流的纯解码器（Decoder-only）大模型。
*   **存储成本**：存储编码器的隐藏状态向量（Embedding）比存储原始文本或压缩索引需要更多的磁盘空间。
*   **语料库规模**：虽然在 1 亿 Token 级别表现良好，但在万亿级（Web-scale）超大规模语料库上的索引效率和检索精度尚待验证。
*   **任务范围**：主要集中在短答案问答，对于长文本生成或摘要任务的表现未做深入探讨。
