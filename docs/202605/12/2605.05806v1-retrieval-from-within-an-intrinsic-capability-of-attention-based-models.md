## 摘要
检索增强生成（RAG）通常将检索和生成视为独立的系统。我们探讨了基于注意力的编码器-解码器是否可以直接从其内部表示中进行检索。我们引入了 INTRA（通过注意力的内在检索），这是一个解码器注意力查询对预编码的证据块进行评分，并将其直接重用为生成上下文的框架。通过这种构建方式，INTRA 统一了检索和生成，消除了 RAG 流水线中典型的检索器-生成器不匹配问题。该设计还通过在不同查询中重用预计算的编码器状态，分摊了上下文编码的开销。在问答基准测试中，INTRA 在证据召回率和端到端回答质量上均优于强大的工程化检索流水线。我们的结果表明，基于注意力的模型已经具备了可以被激发的检索机制，而无需作为外部模块添加。

## Abstract
Retrieval-augmented generation (RAG) typically treats retrieval and generation as separate systems. We ask whether an attention-based encoder-decoder can instead retrieve directly from its own internal representations. We introduce INTRA (INTrinsic Retrieval via Attention), a framework where decoder attention queries score pre-encoded evidence chunks that are then directly reused as context for generation. By construction, INTRA unifies retrieval and generation, eliminating the retriever-generator mismatch typical of RAG pipelines. This design also amortizes context encoding by reusing precomputed encoder states across queries. On question-answering benchmarks, INTRA outperforms strong engineered retrieval pipelines on both evidence recall and end-to-end answer quality. Our results demonstrate that attention-based models already possess a retrieval mechanism that can be elicited, rather than added as an external module.

### 1. 论文的核心问题与整体含义（研究动机和背景）
传统的检索增强生成（RAG）系统通常采用“模块化”设计，将检索器（Retriever）和生成器（Generator）视为两个独立的系统。这种分离导致了两个主要问题：
*   **表示空间不匹配**：检索器通常在向量空间或词频空间操作，而生成器在自身的隐藏表示空间操作，两者之间的信息传递存在“翻译”损耗。
*   **计算冗余**：在标准 RAG 中，检索到的文本需要被生成器重新编码（Re-encoding），这在处理长上下文或多跳推理时会产生巨大的计算开销。

本研究的核心动机是探讨：**基于注意力的模型是否已经具备内在的检索能力？** 作者认为，注意力机制本质上就是一种查询驱动的信息选择过程，因此可以利用模型自身的内部表示直接进行检索，从而实现检索与生成的深度统一。

### 2. 论文提出的方法论：INTRA 框架
作者提出了 **INTRA (INTrinsic Retrieval via Attention)** 框架，其核心思想是让解码器（Decoder）的交叉注意力（Cross-Attention）查询直接对预编码的证据块（Chunks）进行评分。

*   **核心思想**：将检索视为模型内部注意力机制的延伸。不再使用外部检索器，而是利用预训练编码器-解码器模型的内部状态来识别和提取相关信息。
*   **关键技术细节**：
    *   **Reverse-QWK (反向查询键投影)**：这是本文的技术突破点。传统模型中，每一层的 Key 投影是特定的，导致无法共享一个统一的检索索引。INTRA 将层特定的投影移至 Query 端，使得所有层可以共享同一个归一化后的编码器表示池（Shared Pool），极大地降低了存储和计算复杂度。
    *   **可学习检索 Token**：在输入序列中加入少量的可学习 Token（如 64 个），专门用于触发解码器的检索行为，生成用于评分的查询向量。
    *   **后期交互评分 (MaxSim)**：借鉴 ColBERT 的思想，利用解码器查询与编码器块表示之间的最大相似度之和进行块级评分。
    *   **两阶段推理**：第一步进行检索前向传递，获取全局评分并筛选 Top-K 块；第二步直接复用这些块的编码器状态进行答案生成，无需重新编码文本。

### 6. 主要结论与发现
*   **内在检索能力确实存在**：即使是冻结的预训练解码器，通过极少量的参数微调（检索 Token），也能展现出超越专门训练的外部检索器的性能。
*   **多跳场景优势明显**：INTRA 在需要整合多处证据的复杂任务中表现最强，因为它能利用解码器的注意力模式来对齐证据需求。
*   **效率大幅提升**：由于消除了生成阶段的重新编码过程，INTRA 的首字延迟（TTFT）显著低于标准 RAG，且随检索块数量增加的耗时增长更平缓。

### 7. 优点：亮点总结
*   **架构优雅**：实现了检索与生成的真正统一，消除了 RAG 系统中常见的“检索-生成不匹配”问题。
*   **计算高效**：通过状态复用（State Reuse）避免了冗余的编码计算，对长上下文处理非常友好。
*   **Reverse-QWK 创新**：巧妙地解决了 Transformer 多层结构与统一检索索引之间的矛盾。

### 8. 不足与局限
*   **架构限制**：该方法目前仅适用于编码器-解码器（Encoder-Decoder）架构，无法直接应用于目前主流的纯解码器（Decoder-only）大模型。
*   **存储成本**：存储 Token 级的隐藏层表示比存储压缩后的向量索引（如单向量 Embedding）需要更多的磁盘空间。
*   **规模验证**：实验主要在 4B 规模模型和 1 亿 Token 语料库上进行，尚未在万亿级 Token 的超大规模 Web 检索场景下验证其扩展性。
*   **静态语料**：目前的设计更倾向于处理相对静态的知识库，对于频繁更新的动态语料库，索引维护成本可能较高。
