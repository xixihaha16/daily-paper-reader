## 摘要
检索增强生成 (RAG) 在长上下文问答中受限于推理阶段对大规模检索上下文的预填充 (prefilling) 瓶颈。一种常用策略是为单个文档预计算键值 (KV) 缓存，并选择性地重计算一小部分 token 以恢复全局因果依赖，但现有方法依赖于启发式规则或表示差异，而没有建模所选 token 是否能有效影响生成。我们将选择性 KV 重计算建模为一个信息流问题，并证明在推理一致的 RoPE 几何结构下计算时，来自查询 (query) 的简单注意力范数 (attention-norm) 信号可以可靠地识别出既具有语义相关性，又在结构上处于信息传播位置的 token。因此，我们为检索到的分块 (chunks) 重建了全局位置分配，并引入了一种信息流引导的分块重排序策略。在 LLM 和 VLM 基准测试上的实验表明，在相当的效率预算下，该方法相较于先前方法取得了持续的提升。

## Abstract
Retrieval-augmented generation (RAG) for long-context question answering is bottlenecked by inference-time prefilling over large retrieved contexts. A common strategy is to precompute key-value (KV) caches for individual documents and selectively recompute a small subset of tokens to restore global causal dependencies, but existing methods rely on heuristics or representation discrepancies without modeling whether selected tokens can effectively influence generation. We cast selective KV recomputation as an information flow problem and show that a simple attention-norm signal from the query reliably identifies tokens that are both semantically relevant and structurally positioned to propagate information, when computed under an inference-consistent RoPE geometry. We therefore reconstruct global positional assignments for retrieved chunks and introduce an information-flow-guided chunk reordering strategy. Experiments on LLM and VLM benchmarks demonstrate consistent gains over prior methods under comparable efficiency budgets.

### 1. 核心问题与研究动机
*   **核心问题**：在长上下文检索增强生成（RAG）场景下，推理过程中的“预填充（Prefilling）”阶段是主要的性能瓶颈。
*   **研究背景**：为了提高效率，通常会离线预计算单个文档的 KV 缓存。但在推理时，将这些独立计算的缓存拼接后，会丢失跨文档的全局因果依赖（因为每个块是基于局部因果掩码计算的）。
*   **动机**：现有的选择性重计算方法（如 CacheBlend 或 EPIC）要么依赖浅层表示差异，要么使用固定的位置启发式规则，未能准确建模哪些 token 真正能有效影响最终的生成。本文旨在通过“信息流”视角，识别并重计算那些对查询（Query）最具影响力的关键 token。

### 2. 方法论
*   **核心思想**：将选择性 KV 重计算建模为信息流问题，利用查询到上下文 token 的**注意力范数（Attention-Norm）**作为识别信号。
*   **关键技术细节**：
    *   **注意力范数选择**：计算从 Prompt 到上下文 token 的注意力权重之和 $s_j = \sum_{i=1}^M A_{ij}$，选取分值最高的 Top-k 个 token 进行重计算。
    *   **推理一致的 RoPE 几何（Global RoPE）**：发现只有在重建全局位置索引（而非局部索引）的情况下，注意力范数才能准确反映 token 的重要性。
    *   **全局位置重建**：在选择 token 前，将所有检索块按全局顺序排列，并重新分配 RoPE 位置，以确保选择过程与推理时的几何结构一致。
    *   **信息流引导的块重排序**：根据块内关键 token 的重要性对检索到的文档块进行重新排序，将信息量更大的块放置在靠近 Prompt 的位置，以增强信息传播。
    *   **两阶段选择**：在重排序场景下，先用局部 RoPE 进行初步筛选，排序后再用全局 RoPE 进行最终重计算目标锁定。

*   **性能领先**：在相同的重计算预算下，InfoFlow KV 在多跳推理任务上显著优于 CacheBlend 和 EPIC。
*   **几何一致性至关重要**：RoPE 的位置分配直接影响 token 选择的质量，全局位置重建是恢复长程依赖的关键。

### 7. 优点
*   **原理简洁且科学**：从信息流和几何结构出发，避开了复杂的启发式规则。
*   **即插即用**：无需对预训练模型进行任何微调或修改。
*   **硬件友好**：结合序列并行技术，显著降低了长文本 RAG 的首字延迟（TTFT）。
*   **鲁棒性强**：在“大海捞针”实验中表现出极高的稳定性，减少了长文本中的“信息丢失”区域。

### 8. 不足与局限
*   **硬件利用率瓶颈**：由于选择性重计算产生的注意力掩码是不规则的，现有的 FlashAttention 等优化内核无法完美支持，导致实际计算开销高于理论值（约有 2x 的硬件开销）。

*   **应用限制**：对于具有极强内在顺序结构的单体长文档，块重排序策略可能不适用。
