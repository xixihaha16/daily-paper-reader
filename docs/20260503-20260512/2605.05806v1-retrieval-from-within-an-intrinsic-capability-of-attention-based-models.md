## 摘要
检索增强生成（RAG）通常将检索和生成视为独立的系统。我们探讨了基于注意力的编码器-解码器是否可以直接从其自身的内部表示中进行检索。我们引入了 INTRA（通过注意力的内在检索），这是一个解码器注意力查询对预编码的证据块进行评分，并将其直接重用为生成上下文的框架。通过这种构建方式，INTRA 统一了检索和生成，消除了 RAG 流水线中典型的检索器-生成器不匹配问题。该设计还通过在不同查询中重用预计算的编码器状态，分摊了上下文编码的开销。在问答基准测试中，INTRA 在证据召回率和端到端回答质量方面均优于强大的工程化检索流水线。我们的结果表明，基于注意力的模型已经具备了一种可以被激发出来的检索机制，而无需作为外部模块添加。

## Abstract
Retrieval-augmented generation (RAG) typically treats retrieval and generation as separate systems. We ask whether an attention-based encoder-decoder can instead retrieve directly from its own internal representations. We introduce INTRA (INTrinsic Retrieval via Attention), a framework where decoder attention queries score pre-encoded evidence chunks that are then directly reused as context for generation. By construction, INTRA unifies retrieval and generation, eliminating the retriever-generator mismatch typical of RAG pipelines. This design also amortizes context encoding by reusing precomputed encoder states across queries. On question-answering benchmarks, INTRA outperforms strong engineered retrieval pipelines on both evidence recall and end-to-end answer quality. Our results demonstrate that attention-based models already possess a retrieval mechanism that can be elicited, rather than added as an external module.

### 1. 核心问题与整体含义（研究动机和背景）
传统的检索增强生成（RAG）架构通常采用“模块化”设计，将检索器（Retriever）和生成器（Generator）视为两个独立的系统。这种分离导致了两个主要问题：
*   **表示空间不匹配**：检索器使用的嵌入空间（如向量数据库）与生成器处理文本的隐空间不一致，导致检索到的信息未必是生成器最需要的。
*   **计算冗余**：在标准 RAG 中，检索到的文本需要被生成器重新编码（Re-encoding），这在处理长上下文或高频查询时会产生巨大的计算开销。

**研究动机**：作者提出，注意力机制（Attention）本质上就是一种基于查询的信息选择和加权机制。因此，他们探索是否可以利用预训练编码器-解码器（Encoder-Decoder）模型的**内在注意力机制**来直接执行检索任务，从而实现检索与生成的深度统一。

### 2. 方法论：INTRA 框架
作者提出了 **INTRA (INTrinsic Retrieval via Attention)** 框架，其核心思想是让解码器的交叉注意力（Cross-Attention）查询直接对预编码的证据块进行评分和提取。

*   **核心思想**：将检索视为解码器的一种内在行为。编码器一次性将语料库预编码为向量，解码器通过特定的“检索查询”在向量空间中筛选信息，选中的向量直接进入解码阶段，无需转回文本再重新编码。
*   **关键技术细节**：
    *   **Reverse-QWK (逆向查询-键投影)**：这是本文的技术核心。传统模型每层都有独立的 Key 投影，导致无法共享一个统一的检索索引。Reverse-QWK 将 Key 侧的缩放和投影操作移至 Query 侧，使得所有层可以共享同一个标准化的编码器表示池（Shared Pool），极大降低了存储和检索复杂度。
    *   **可学习的检索 Token ($\rho$)**：在解码器输入中加入少量的可学习 Token，专门用于诱导解码器产生能够代表检索需求的查询向量。
    *   **MaxSim 评分机制**：借鉴 ColBERT 的后期交互（Late Interaction）思想，利用解码器查询状态与编码器块表示之间的最大相似度之和进行块级评分。
    *   **两阶段推理流程**：
        1.  **检索阶段**：解码器进行一次快速前向传播，生成检索查询并对全库评分，选出 Top-k 个证据块。
        2.  **生成阶段**：直接复用这 Top-k 个块的编码器隐状态（Hidden States）作为交叉注意力的上下文，生成最终答案。

### 6. 主要结论与发现
*   **检索性能优异**：在多跳问答任务中，INTRA 的完整证据召回率显著超过了专门训练的稠密检索器（如 BGE 和 Qwen3）。
*   **端到端质量提升**：在 HotPotQA 等数据集上，INTRA 带来的端到端答案准确率（EM）比强力 RAG 基准高出约 4-5 个百分点。
*   **效率优势**：由于无需在生成阶段重新编码检索到的文本，INTRA 的首字延迟（TTFT）在检索块数量增加时保持稳定，而传统 RAG 的延迟随 $k$ 值呈二次方增长。
*   **内在一致性**：实验证明，当检索器和生成器共享同一个解码器逻辑时，模型能更好地闭合“检索-生成”之间的鸿沟。

### 7. 优点与亮点
*   **架构优雅**：将检索从“外部插件”转变为“内在能力”，简化了 RAG 的系统复杂度。
*   **计算高效**：通过 Reverse-QWK 实现了编码器状态的跨查询复用，极大地优化了预填充（Prefilling）阶段的性能。
*   **无需大规模重训**：仅需微调极少量参数即可激发预训练模型的检索潜力，证明了注意力机制的通用性。

### 8. 不足与局限
*   **架构限制**：该方法目前仅适用于 **Encoder-Decoder** 架构（如 T5, Bart），无法直接应用于目前主流的纯 Decoder 模型（如 Llama, GPT 系列），因为后者缺乏显式的交叉注意力机制。
*   **存储开销**：虽然计算省了，但存储预编码的向量（隐状态）比存储原始文本或压缩后的嵌入向量占用更多空间（10 亿 Token 约需 2.5TB 存储）。
*   **语料库规模验证**：实验语料库为 1 亿 Token 级别，尚未在万亿级（Web-scale）超大规模索引上验证其检索效率和冲突率。
*   **任务局限**：主要集中在短答案问答，对于长文本摘要或开放式对话的检索表现尚待观察。
