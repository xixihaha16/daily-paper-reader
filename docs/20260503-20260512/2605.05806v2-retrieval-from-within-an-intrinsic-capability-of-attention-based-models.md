## 摘要
检索增强生成（RAG）通常将检索和生成视为独立的系统。我们探讨了基于注意力的编码器-解码器是否可以直接从其自身的内部表示中进行检索。我们引入了 INTRA（通过注意力的内在检索），这是一个解码器注意力查询对预编码的证据块进行评分，并将其直接重用为生成上下文的框架。通过这种构建方式，INTRA 统一了检索和生成，消除了 RAG 流水线中典型的检索器-生成器不匹配问题。该设计还通过在不同查询中重用预计算的编码器状态，分摊了上下文编码的开销。在问答基准测试中，INTRA 在证据召回率和端到端回答质量方面均优于强大的工程化检索流水线。我们的结果表明，基于注意力的模型已经具备了一种可以被诱导出的检索机制，而无需作为外部模块添加。

## Abstract
Retrieval-augmented generation (RAG) typically treats retrieval and generation as separate systems. We ask whether an attention-based encoder-decoder can instead retrieve directly from its own internal representations. We introduce INTRA (INTrinsic Retrieval via Attention), a framework where decoder attention queries score pre-encoded evidence chunks that are then directly reused as context for generation. By construction, INTRA unifies retrieval and generation, eliminating the retriever-generator mismatch typical of RAG pipelines. This design also amortizes context encoding by reusing precomputed encoder states across queries. On question-answering benchmarks, INTRA outperforms strong engineered retrieval pipelines on both evidence recall and end-to-end answer quality. Our results demonstrate that attention-based models already possess a retrieval mechanism that can be elicited, rather than added as an external module.

### 1. 核心问题与整体含义（研究动机和背景）
传统的检索增强生成（RAG）系统通常采用“模块化”设计，将**检索器（Retriever）**和**生成器（Generator）**视为两个独立的组件。这种架构存在两个主要痛点：
*   **表示空间不匹配（Mismatch）：** 检索器（如基于向量或词频）与生成器（大语言模型）在不同的表示空间运行，检索到的内容未必是生成器最需要的。
*   **计算冗余：** 在标准 RAG 中，检索到的文本需要被生成器重新编码（Re-encoding），这在处理长上下文或大规模证据时会产生显著的计算延迟。

**研究动机：** 作者提出，注意力机制本质上就是一种查询驱动的信息选择机制。因此，他们探索是否可以利用预训练编码器-解码器（Encoder-Decoder）模型的**内在注意力机制**直接进行检索，从而实现检索与生成的深度统一。

### 2. 方法论：INTRA 框架
论文提出了 **INTRA (INTrinsic Retrieval via Attention)** 框架，其核心思想是让解码器的交叉注意力查询（Cross-Attention Queries）直接对预先编码好的证据块进行评分。

*   **核心思想：** 将检索视为模型内部的一种“远程注意力”行为。预先使用编码器将语料库处理成向量并存储，解码器在回答问题时，通过其注意力头产生的 Query 与这些存储的向量进行匹配。
*   **关键技术细节：**
    *   **可学习的检索 Token（Retrieval Tokens）：** 在解码器输入中加入少量的特殊 Token，专门用于诱导产生检索所需的查询向量。
    *   **MaxSim 评分机制：** 借鉴 ColBERT 的思想，计算查询 Token 与证据块 Token 之间的最大相似度之和，作为检索评分。
    *   **Reverse-QWK（反向查询-键投影）：** 这是本文的技术亮点。为了解决不同层 Key 投影矩阵不一致导致无法复用向量的问题，作者将 Key 端的线性变换和缩放“反向”移动到 Query 端。这样，整个语料库只需要存储一份标准化的编码器表示，所有层都可以共享同一个索引进行检索。
*   **流程：** 
    1.  **初始筛选（S0）：** 基于简单的向量相似度初步筛选候选块。
    2.  **检索 Pass：** 解码器运行一次前向传播，利用检索 Token 生成查询向量，对语料库进行精确评分。
    3.  **生成 Pass：** 将 Top-k 证据块的预编码向量直接喂给解码器，生成最终答案。

### 3. 实验设计
*   **数据集：** 涵盖了四种维基百科问答基准：
    *   **多跳推理：** HotPotQA, 2WikiMultihopQA, MuSiQue（需要整合多处证据）。
    *   **单跳问答：** Natural Questions (NQ)。
*   **语料库规模：** 构建了一个包含约 75.9 万个文本块、总计 1 亿 Token 的共享检索池。
*   **对比方法（Baselines）：**
    *   **传统检索：** TF-IDF, BM25。
    *   **稠密检索：** BGE-large, Qwen3-Embedding (0.6B/4B)。
    *   **重排序与混合：** Jina Reranker, Hybrid RAG (RRF 融合)。
    *   **生成器：** 统一使用 T5Gemma2-4B 作为生成底座以确保公平。

### 4. 资源与算力
*   **硬件：** 使用了 NVIDIA H100 (80GB) GPU。
*   **算力消耗：**
    *   **QA 预训练阶段：** 使用多达 160 张 GPU，耗时约 12 小时（约 2000 GPU 小时）。
    *   **检索/生成微调：** 每个任务的训练耗时小于 200 GPU 小时。
*   **存储：** 论文提到 10 亿 Token 的语料库在 8-bit 量化下约需 2.5TB 存储，可存放在单个 NVMe SSD 中。

### 5. 实验数量与充分性
*   **实验规模：** 论文在 4 个主流数据集上进行了完整的检索召回率（Recall@k）和端到端 QA 性能（EM/F1）测试。
*   **消融实验：** 非常充分。作者测试了初始上下文 S0 的影响、检索 Token 的数量（1 vs 16 vs 64）、块池化长度（Lp）以及不同生成器底座的兼容性。
*   **客观性：** 实验对比了当前最强的开源检索模型（如 Qwen3-Embedding），并提供了 95% 的置信区间，结果具有较高的可信度。

### 6. 主要结论与发现
*   **多跳能力卓越：** INTRA 在需要证据组装的多跳 QA 任务上显著优于所有基线模型，证明了解码器注意力能更好地识别复杂的证据关联。
*   **内在能力：** 实验证明，无需引入外部复杂的检索模型，仅通过轻量级微调（仅 16.4 万个参数），就能激发预训练模型自带的检索潜力。
*   **效率提升：** 由于跳过了生成阶段的证据重编码过程，INTRA 的首字延迟（TTFT）在检索大量证据块时比传统 RAG 快一个数量级。

### 7. 优点与亮点
*   **架构优雅：** 实现了检索与生成的真正统一，消除了“检索器不知道生成器想要什么”的根本矛盾。
*   **计算复用：** Reverse-QWK 技术巧妙地解决了 Transformer 层间参数不一致的问题，使得预编码向量可以跨查询、跨层复用。
*   **性能强劲：** 在不使用大规模检索专用数据预训练的情况下，其检索效果达到了甚至超过了专门的检索模型。

### 8. 不足与局限
*   **架构限制：** 该方法目前仅适用于 **Encoder-Decoder** 架构（如 T5 系列），而当前主流的 LLM 多为 Decoder-only 架构，如何迁移仍是挑战。
*   **存储开销：** 存储 Token 级别的隐藏层向量（Embedding）比存储原始文本或压缩后的特征向量需要更多的磁盘空间。
*   **语料库动态性：** 论文主要针对固定知识库，对于频繁更新的动态语料库，维护预编码索引的成本可能较高。
*   **单跳任务优势不明显：** 在简单的单跳任务（如 NQ）中，INTRA 相比于强力外部检索器的领先优势较小。
