## 摘要
检索增强生成 (RAG) 通过将生成过程锚定在外部知识上，缓解了大语言模型 (LLM) 的幻觉问题。然而，标准的 RAG 方法在多跳推理方面面临挑战。虽然近期基于图的 RAG 方法改善了互连文本块的检索，但它们往往依赖于计算成本高昂且易出错的基于 LLM 的提取流水线。为解决这些问题，我们提出了 TIGRAG (Token-Induced GraphRAG)，这是一种基于 Token 共现知识图谱的高效图增强 RAG 框架。TIGRAG 利用滑动窗口共现统计直接建模 Token 之间的拓扑关系，从而实现了可扩展的图构建。在推理阶段，它结合了基于图的语义扩展和神经重排序，以检索用于多跳推理的互连证据。具体而言，它引入了一种迭代的实体驱动检索策略，利用从先前检索到的上下文中提取的桥接实体逐步扩展查询。我们在三个广泛采用的多跳问答 (QA) 基准测试上评估了 TIGRAG。实验结果表明，我们的框架在检索和下游问答任务中均持续优于稠密检索和基于图的 RAG 方法，同时大幅减少了索引时间、推理延迟和提示词占用。

## Abstract
Retrieval-Augmented Generation (RAG) mitigates hallucinations in Large Language Models (LLMs) by grounding the generation process on external knowledge. However, standard RAG approaches struggle with multi-hop reasoning. While recent graph-based RAG methods improve the retrieval of interconnected chunks, they often rely on computationally expensive and error-prone LLM-based extraction pipelines. To address these issues, we propose TIGRAG (Token-Induced GraphRAG), an efficient graph-augmented RAG framework based on a token co-occurrence Knowledge Graph. TIGRAG directly models topological relationships between tokens using sliding-window co-occurrence statistics, thus enabling scalable graph construction. During inference, it combines graph-based semantic expansion and neural reranking to retrieve interconnected evidence for multi-hop reasoning. Specifically, it introduces an iterative entity-driven retrieval strategy that progressively expands the query using bridging entities extracted from previously retrieved contexts. We evaluated TIGRAG on three widely adopted multi-hop Question Answering (QA) benchmarks. Experimental results demonstrated that our framework consistently outperforms dense retrieval and graph-based RAG methods in both retrieval and downstream QA tasks, while substantially reducing indexing time, inference latency, and prompt footprint.

### 1. 核心问题与研究动机
*   **核心问题**：传统的检索增强生成（RAG）在处理需要跨多个文档合成信息的“多跳推理”任务时表现不佳。
*   **研究背景**：
    *   **标准 RAG**：将文档视为孤立单元，难以捕捉跨文档的语义关联。
    *   **现有 GraphRAG**：虽然引入了图结构，但大多依赖大语言模型（LLM）进行实体和关系的提取。这导致了**高昂的计算成本**、**显著的索引延迟**以及由于 LLM 幻觉导致的**图结构错误传播**。
*   **研究目标**：开发一种既能处理复杂多跳查询，又具备极高构建效率和低推理延迟的图增强 RAG 框架。

### 2. 方法论：TIGRAG 框架
TIGRAG 的核心思想是**放弃昂贵的 LLM 提取过程，转而利用统计学方法构建 Token 级别的共现图**。

*   **离线图构建**：
    *   **文本分块**：使用滑动窗口策略将语料库切分为重叠的文本块（Chunks）。
    *   **Token 共现图**：提取每个块中的有效 Token（去除停用词、短词等），在滑动窗口内统计 Token 的共现频率。节点代表 Token，边权重代表共现频率。
    *   **索引**：同时预计算文本块的向量嵌入（Embeddings）和 Token 到文本块的映射表。
*   **在线推理流程**：
    1.  **语义扩展**：利用个性化 PageRank (PPR) 算法，以查询中的 Token 为种子节点，在图中传播并识别语义相关的扩展 Token。
    2.  **图加权评分**：结合 BM25 概率检索与 PPR 拓扑权重，对候选文本块进行初步排序。
    3.  **动态池化与重排序**：通过累积得分阈值动态筛选候选池，随后使用神经重排序（向量相似度）精选出最相关的文本块。
    4.  **迭代式实体驱动检索**：针对多跳查询，利用命名实体识别 (NER) 从首轮检索结果中提取新实体，将其拼接到原始查询中进行下一轮检索，从而发现隐藏的关联证据。

### 6. 主要结论与发现
*   **性能领先**：TIGRAG 在所有多跳数据集上的检索和生成指标均优于现有方法。例如，在 HotpotQA 上，其 R@2 比 NaiveRAG 高出 9.7%。
*   **效率飞跃**：
    *   **索引速度**：比基于 LLM 的 GraphRAG 快几个数量级（从数百小时缩短至几分钟）。
    *   **推理延迟**：由于检索到的上下文更精准、冗余更少，其端到端推理速度在复杂任务上甚至超过了 NaiveRAG。
    *   **Prompt 压缩**：生成的上下文长度显著短于 LightRAG 等方法，降低了 LLM 的处理负担。
*   **多跳有效性**：迭代式的实体驱动策略能有效填补查询与远端证据之间的语义鸿沟。

### 7. 优点与亮点
*   **无需 LLM 建图**：这是最大的创新点，通过统计学 Token 共现代替 LLM 提取，极大地降低了落地门槛和成本。
*   **拓扑与语义结合**：巧妙地将图论算法（PPR）与现代神经检索（Embedding）结合，兼顾了结构关联和语义理解。
*   **动态性**：引入动态候选池切分，避免了固定 Top-K 带来的噪声或信息缺失。

### 8. 不足与局限
*   **NER 依赖性**：多跳扩展高度依赖 NER 质量，若实体识别出错，可能导致检索偏离目标（语义漂移）。
*   **共现关系的局限**：Token 共现主要捕捉局部邻近性，对于文档中跨度极大或隐含的逻辑关系，单纯的统计共现可能不如语义关系建模深刻。
*   **模态限制**：目前仅限于纯文本处理，尚未扩展到多模态或包含复杂表格的数据场景。
*   **Token 歧义**：将 Token 作为节点可能面临一词多义（Polysemy）问题，文中提到这是未来的改进方向。
