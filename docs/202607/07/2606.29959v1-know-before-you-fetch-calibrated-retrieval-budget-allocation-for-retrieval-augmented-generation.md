## 摘要
检索增强生成（RAG）通常为每个查询检索固定数量的段落。当阅读器已经知道答案时，这是一种浪费；而当不相关或部分相关的段落干扰阅读器时，这可能是有害的。我们将自适应 RAG 形式化为校准的检索预算分配：给定一个查询，决定是进行闭卷回答、检索紧凑上下文（k=1）、检索完整上下文（k=5）还是弃权。本文的贡献在于提供了一个概率接口，而非一种新的原始不确定性信号。我们将序列对数概率和前缀 Logit 不确定性信号校准为正确性概率，然后将这些概率用于分级上下文选择、选择性弃权以及显式的延迟/Token 权衡。在 TriviaQA、Natural Questions 和 MS MARCO 的核心问答实验中，结合辅助的 PopQA 动机和 Qwen/Llama 系列验证，诊断性的折外校准显著提升了概率质量：对于序列对数概率，TriviaQA 上的 ECE 从 0.275 降至 0.062，NQ 上从 0.643 降至 0.009，MS MARCO 上从 0.711 降至 0.031。分级检索改善了我们的信号以及 TARG 风格的前缀熵/边际的完整上下文和段落预算前沿，而检索调用 AUC 与二元门控基本持平，因为 k=1 仍然是一次检索调用。留出训练/验证/测试阈值实验报告了可部署的工作点。在精度匹配的前沿工作点上，实测成本模型显示门控并非在所有情况下都更快：它在 Qwen3-8B 上增加了约 27% 的延迟，但在 Qwen3-32B 上节省了约 8%。这些结果支持了对自适应 RAG 的细致看法：校准置信度最好被理解为在任务和系统约束下分配检索预算的可重用接口。

## Abstract
Retrieval-augmented generation (RAG) typically retrieves a fixed number of passages for every query. This is wasteful when the reader already knows the answer, and it can be harmful when irrelevant or partially relevant passages distract the reader. We formulate adaptive RAG as calibrated retrieval-budget allocation: given a query, decide whether to answer closed-book, retrieve a compact context (k=1), retrieve a full context (k=5), or abstain. The contribution is a probability interface rather than a new raw uncertainty signal. We calibrate sequence log-probability and prefix-logit uncertainty signals into probabilities of correctness, then use these probabilities for graded context selection, selective abstention, and explicit latency/token trade-offs. Across core QA experiments on TriviaQA, Natural Questions, and MS MARCO, with auxiliary PopQA motivation and Qwen/Llama family checks, diagnostic out-of-fold calibration improves probability quality dramatically: for sequence log-probability, ECE drops from 0.275 to 0.062 on TriviaQA, 0.643 to 0.009 on NQ, and 0.711 to 0.031 on MS MARCO. Graded retrieval improves full-context and passage-budget frontiers for both our signal and TARG-style prefix entropy/margin, while retrieval-call AUC remains essentially tied with binary gating because k=1 is still a retrieval call. Held-out train/validation/test threshold experiments report deployable operating points. At matched-accuracy frontier operating points, a measured cost model reveals that gating is not universally faster: it increases latency by about 27% on Qwen3-8B but saves about 8% on Qwen3-32B. These results support a nuanced view of adaptive RAG: calibrated confidence is best understood as a reusable interface for allocating retrieval budget under task and system constraints.

传统的检索增强生成（RAG）系统通常对所有查询执行固定数量的文档检索。这种做法存在两个主要弊端：一是**资源浪费**，当模型自身参数记忆已包含答案时，检索会增加不必要的延迟和计算成本；二是**性能干扰**，引入无关或部分相关的文档可能误导模型，导致原本正确的闭卷回答变错。
本文的核心动机是将自适应 RAG 视为一个**校准后的检索预算分配问题**。研究者旨在建立一个概率接口，让系统在生成前预知风险，从而动态决定是直接回答（k=0）、检索少量信息（k=1）、检索完整上下文（k=5）还是直接放弃回答（弃权）。

### 2. 核心方法论
论文提出了一种轻量级的概率校准框架，其核心步骤如下：
*   **不确定性信号提取**：首先让冻结的阅读器模型在无检索状态下生成一个简短答案，提取其序列对数概率（Sequence Log-probability）或前缀逻辑值（Prefix-logit）等原始不确定性指标。
*   **逻辑校准（Calibration）**：使用逻辑回归（Logistic Calibrator）将原始标量信号映射为模型闭卷回答正确的概率 $P(correct)$。这一步将“原始分数”转化为具有物理意义的“置信度”。
*   **分级分配策略**：根据校准后的概率设定多个阈值，将查询分配至不同的动作：
    *   高置信度 $\rightarrow$ 闭卷回答（跳过检索）。
    *   中置信度 $\rightarrow$ 紧凑检索（k=1）。
    *   低置信度 $\rightarrow$ 完整检索（k=5）。
    *   极低置信度 $\rightarrow$ 弃权（Abstain）。
*   **成本模型**：引入了一个考虑闭卷探测开销、检索开销和生成开销的实测成本公式，用于评估门控机制在实际部署中的盈亏平衡点。

### 6. 主要结论与发现
*   **校准至关重要**：原始信号的校准误差（ECE）极高，校准后在 NQ 等数据集上 ECE 可从 0.643 降至 0.009，显著提升了决策的可靠性。

*   **模型规模效应**：模型越大，闭卷准确率越高，可跳过检索的空间就越大，自适应 RAG 的收益越明显。

### 7. 优点
*   **接口化思维**：将不确定性转化为概率接口，使得不同模型、不同任务之间的决策标准可以统一。
*   **非侵入性**：无需对模型进行微调，适用于现有的冻结 LLM 和检索器。
*   **成本意识**：不仅关注准确率，还深入分析了实际部署中的延迟和 Token 成本，具有很强的工程指导意义。

### 8. 不足与局限
*   **任务局限性**：实验主要集中在短答案问答（Short-answer QA），对于长文本生成或需要多步推理的任务，校准效果和决策逻辑尚待验证。
*   **硬件敏感性**：延迟分析基于特定硬件和 Batch-1 设置，在生产环境（如高并发、KV 缓存复用）下的表现可能有所不同。
*   **分布偏移风险**：校准器依赖于验证集的分布，如果部署时面临严重的领域偏移（Domain Shift），校准概率可能会失效。
